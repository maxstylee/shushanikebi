# Shushanikebi - Front-End Exam Project
### Bound in body, boundless in soul

## Overview

This project is a front-end web application developed as an exam assignment. It demonstrates practical knowledge of HTML5, CSS3, SCSS, and Git/GitHub workflow. The layout is implemented based on a **[Figma design](https://www.figma.com/design/jrlUWJdbxTc2bKd5dpRa5r/10X-markup-exam-project---Spotify?node-id=3-2&t=a62aCfDedWMpgRI0-1)** and follows a clean structure, semantic markup, and organized styling architecture.

## Team Decisions

### Development Approach
- The project was developed using a **Desktop-First approach**.  
  We initially focused on building a well-structured layout for larger screens and then adapted the design for smaller devices and mobile responsiveness.

### Task Management
- We used **Google Docs** to organize and track tasks.  
- Responsibilities were clearly divided between team members, and tasks were **planned and assigned in advance** to ensure a structured workflow.

### Team Communication
- The team maintained **regular synchronization through short and frequent meetings**, where we reviewed progress, discussed challenges, and coordinated the next steps.

### Naming Convention
- We followed the **BEM (Block Element Modifier)** naming methodology for CSS classes.  
- This approach helped maintain **consistent, scalable, and easily readable code**, making the structure of styles clearer across the project.

---

## Workflow
- Development was organized using **separate Git branches** for different tasks and features.
- Changes were integrated into the main branch through **Pull Requests**, allowing for review and better version control.
- Each team member was **responsible for their assigned parts of the project**, ensuring clear ownership and efficient collaboration.
